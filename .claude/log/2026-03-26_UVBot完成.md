## 技術設計

### API 資料來源

- **API**: 中央氣象署 O-A0005-001（紫外線指數每日最大值）
- **回傳格式**: `records.weatherElement.location = [{StationID, UVIndex}, ...]`（共 30 站）
- **問題**: API 只有 StationID，無縣市欄位

### StationID → 縣市 靜態對應表

CWA O-A0005-001 的 StationID 與其他氣象 API 測站系統不同，無法交叉查詢。
→ 解決方案：手動建立 `stationCountyMap.js`，涵蓋台灣主要縣市 24 個測站。

**排除站（避免海拔偏差）**：
- 鞍部 `467350`（825m）
- 阿里山 `466920`（2413m）
- 玉山 `467590`（3845m）
- 彭佳嶼 `467441`（孤立離島）
- 竹子湖 `467270`（607m）

### 縣市查詢策略

```
用戶城市
  → CITY_ALIAS 轉換（苗栗 → 新竹市 等 6 組）
  → STATION_COUNTY_MAP 找對應測站
  → 多站取平均（如高雄市：高雄 + 岡山 2 站）
  → 無對應站 → fallback 全台已知測站平均
```

**無直屬測站縣市（CITY_ALIAS）**：
| 查詢 | 借用 |
